# TempRef — Temporal Refinement for Video Super-Resolution

Due to the large sizes of the files, we could only upload them as a release v1.0. It contains the code, trained models, and generated outputs for **TempRef**, a lightweight post-processing network that reduces temporal flickering in the outputs of frame-based video super-resolution models. The project is described in full in our report.

### Data (`data/`, `val_sharp/`)

All folders whose names begin with `data` or `val_sharp` contain video frame data. This includes the REDS validation set (HR ground truth and pre-computed Real-ESRGAN outputs used as refiner inputs), as well as the refined output frames produced during evaluation.

### Code (`code/`)

The `code/` folder contains all Jupyter notebooks used in the project:

- `precompute_esrgan.ipynb` — runs Real-ESRGAN on the LR input frames and caches the outputs
- `dataset_dev.ipynb` — dataset class development and verification
- `train.ipynb`, `train_v2.ipynb`, ..., `train_v4.ipynb` — training notebooks for each loss variant (V1 through V4)
- `train_new_loss.ipynb` — motion-compensated feature agreement loss experiments
- `train_efrlfn_loss.ipynb` — EfRLFN-inspired composite loss experiments
- `evaluation.ipynb`, `evaluation_v2.ipynb` — quantitative evaluation on REDS4 (PSNR, LPIPS, temporal consistency error)

### Checkpoints (`checkpoints*/`)

All folders whose names begin with `checkpoints` contain saved model weights. Intermediate checkpoints are saved every 1000 iterations during training; the best checkpoint (by rolling-average training loss) is saved separately as `refiner_*_best.pth`. To run inference with a trained model, load the `_best.pth` file from the relevant folder.

### Videos (`videos*/`)

All folders whose names begin with `videos` contain comparison videos generated by our models. Each clip is rendered as a side-by-side comparison (ground truth / vanilla Real-ESRGAN / refined output) for visual evaluation of temporal consistency.

## Reproducing Results

1. Run `precompute_esrgan.ipynb` to generate the Real-ESRGAN outputs used as refiner inputs.
2. Run the desired `train_*.ipynb` notebook to train a refiner variant.
3. Run the corresponding `evaluation*.ipynb` notebook to compute PSNR, LPIPS, and TCE on REDS4.

All notebooks are designed to run on Google Colab with a GPU runtime. Data is read from and checkpoints are written to Google Drive; update the path variables at the top of each notebook to match your Drive layout.
